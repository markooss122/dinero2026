Herramientas y recursos que realmente uso

En esta página recopilo herramientas que considero útiles para montar y gestionar proyectos online: desde desarrollo y organización hasta creación de contenido y trabajo con APIs. No necesitas usar todas; lo importante es escoger pocas y exprimirlas bien.

Nota: algunos enlaces pueden ser externos. Revisa siempre las condiciones de cada herramienta (versiones gratuitas, precios, política de datos, etc.) antes de usarlas en tu proyecto.

Desarrollo y control de versiones

Imprescindibles para construir cualquier proyecto web de forma ordenada y profesional.

  • GitHub

    Categoría: Código y repositorios | Nivel: Intermedio

    Plataforma de desarrollo colaborativo donde puedes guardar tu código, gestionar versiones, trabajar con ramas y compartir proyectos públicos o privados. Ideal para tener copias de seguridad y un historial claro de cambios.

    Cómo lo uso: alojar proyectos, compartir código con otras personas y documentar ideas de negocio o pruebas técnicas.

    Visitar GitHub

  • Git

    Categoría: Control de versiones | Nivel: Intermedio

    Sistema de control de versiones distribuido. Permite llevar registro de todos los cambios en tus archivos, volver atrás si algo sale mal y trabajar en paralelo en diferentes funcionalidades.

    Recomendación: aunque al principio parece complejo, aprender los comandos básicos te ahorra muchos problemas cuando tu proyecto crece.

    Visitar página de Git

  • Visual Studio Code

    Categoría: Editor de código | Nivel: Principiante‑Intermedio

    Editor de código ligero pero muy potente, con extensiones para JavaScript, HTML, CSS, Python y muchos más lenguajes. Incluye integración con Git, terminal integrada y soporte para depuración.

    Truco: instalar extensiones como linting, formateo automático y snippets de HTML/CSS ayuda a escribir código más limpio y rápido.

    Visitar Visual Studio Code

Documentación, notas y formato

Para documentar procesos, ideas de negocio y guías de forma clara.

  • Markdown

    Categoría: Documentación | Nivel: Principiante

    Lenguaje de marcado ligero que permite dar formato al texto (títulos, listas, enlaces, tablas) sin depender de editores visuales. Se usa muchísimo en documentación técnica, GitHub y blogs estáticos.

    Idea: puedes usar Markdown para escribir guías internas de tu proyecto, resúmenes de cursos o documentación de tus sistemas para generar ingresos.

    Visitar Markdown Guide

APIs y automatización básica

Herramientas útiles si quieres conectar servicios entre sí o probar integraciones.

  • Postman

    Categoría: APIs y testing | Nivel: Intermedio

    Aplicación para enviar y probar peticiones a APIs. Te permite ver fácilmente las respuestas, guardar colecciones de peticiones y documentar integraciones. Muy útil si quieres automatizar partes de tu negocio o consumir datos de otros servicios.

    Ejemplo de uso: probar el endpoint de una pasarela de pago, un servicio de email o una API de IA antes de integrarla en tu web.

    Visitar Postman

Comunidades y resolución de dudas

Cuando te atascas con código o con la parte técnica, estas comunidades ayudan muchísimo.

  • Stack Overflow

    Categoría: Comunidad técnica | Nivel: Todos

    Comunidad de preguntas y respuestas orientada a programación y desarrollo. Es muy útil para buscar errores concretos, ejemplos de código y buenas prácticas.

    Consejo: antes de preguntar, busca si tu duda ya fue resuelta y recuerda aportar contexto (qué intentaste, qué error exacto sale, etc.).

    Visitar Stack Overflow
